Chapter 01 · Foundations

Introduction to DSPy

DSPy is a framework for algorithmically optimizing LLM prompts and weights — replacing hand-crafted prompt strings with declarative, composable modules that can be automatically tuned using a metric-driven compiler.

PROJECT THEME

Evaluation-Driven RAG Pipeline Using DSPy

This documentation walks through a complete, production-grade DSPy project: a Retrieval-Augmented Generation system with multi-step reasoning, automated prompt optimization, and a rigorous evaluation framework — built for enterprise LLM engineering teams.

What is DSPy?

DSPy (Declarative Self-improving Language Programs) is a research-to-production framework developed by the Stanford NLP Group. Unlike prompt engineering frameworks that treat prompts as static strings, DSPy treats LLM pipelines as programs — with learnable parameters, composable modules, and systematic optimization.

At its core, DSPy makes three foundational shifts in how we build LLM applications:

🔧
Programs, Not Prompts
Define what your pipeline should accomplish using typed signatures. Let the optimizer figure out how to instruct the model.
📐
Compilation Over Hand-Tuning
DSPy compilers (optimizers) synthesize few-shot examples and instructions automatically by running your metric on a small training set.
🔁
Evaluation-Driven Development
Every optimization loop is anchored to a measurable metric. You iterate on the metric, not on the prompt text.
🧩
Modular by Default
Swap LLM backends, retrieval models, or reasoning strategies without rewriting your pipeline logic.
🏭
Production Ready
Optimized programs are serializable, versionable, and reproducible — critical for enterprise governance and compliance.
🧠
Multi-Model Support
Works with OpenAI, Anthropic, Cohere, Mistral, local Ollama models, and any LiteLLM-compatible backend.

The Philosophy

Traditional LLM engineering involves an exhausting loop: write a prompt → test it → find an edge case → patch the prompt → break another case → repeat. This process is:

  • Brittle: Prompt changes break downstream behavior unpredictably
  • Unscalable: Expert prompt engineers are a bottleneck
  • Unversioned: Prompt diffs are hard to reason about
  • Non-transferable: A GPT-4 prompt rarely works on Llama-3

DSPy replaces this with a principled, ML-style development loop:

// DSPy Development Loop
 ┌─────────────────────────────────────────────────────────────────────────┐
 │                      DSPy DEVELOPMENT LOOP                              │
 │                                                                         │
 │  1. DEFINE        →   Write Signatures (typed input/output contracts)   │
 │  2. COMPOSE       →   Assemble Modules (Chain, RAG, CoT, ReAct)         │
 │  3. EVALUATE      →   Build dev/test sets + define metric function      │
 │  4. OPTIMIZE      →   Run compiler (MIPROv2, BootstrapFewShot, etc.)    │
 │  5. VALIDATE      →   Score on held-out test set                        │
 │  6. DEPLOY        →   Serialize + serve optimized program               │
 │  7. MONITOR       →   Log traces, detect drift, re-optimize             │
 │                                                                         │
 │          ◄──────────────────────── ITERATE ──────────────────────────   │
 └─────────────────────────────────────────────────────────────────────────┘

Quick Taste: Hello DSPy

PYTHON · hello_dspy.py
import dspy

# 1. Configure the language model
lm = dspy.LM('openai/gpt-4o-mini', api_key='...')
dspy.configure(lm=lm)

# 2. Define a Signature (input → output contract)
class QA(dspy.Signature):
    """Answer questions accurately and concisely."""
    question: str = dspy.InputField(desc="The question to answer")
    answer: str = dspy.OutputField(desc="A factual, concise answer")

# 3. Create a Module (wraps the signature with a reasoning strategy)
qa_module = dspy.ChainOfThought(QA)

# 4. Run inference
result = qa_module(question="What is the capital of France?")
print(result.answer)  # → "Paris"
print(result.rationale)  # → "The capital of France is Paris, ..."
💡 Key Insight

Notice there is no prompt string in the code above. DSPy synthesizes the actual prompt from the Signature's type annotations, field descriptions, and (after optimization) few-shot examples. This is the fundamental shift DSPy introduces.

Chapter 02 · Motivation

Why DSPy Exists

The LLM engineering space had a dirty secret: we were building production systems on top of handcrafted strings. DSPy was born to solve the structural fragility of prompt-centric development at scale.

The Problem with Prompt Engineering

As organizations scaled LLM deployments, a set of recurring failure modes emerged. These weren't bugs — they were systemic limitations of prompt-centric development:

1
The Brittleness Problem
A prompt optimized for GPT-4 fails on Claude. A prompt that works for query type A breaks for query type B. Small wording changes cause cascading behavioral shifts. Prompt engineering is inherently brittle because it relies on model-specific, distribution-specific string hacks.
2
The Evaluation Vacuum
Most prompt engineering is evaluated by human "vibes." When your evaluation loop is subjective, you cannot systematically improve. Teams spend weeks tuning prompts only to discover regressions in prod that weren't caught in testing.
3
The Scalability Wall
Expert prompt engineers are rare and expensive. As pipeline complexity grows (multi-hop RAG, agents, tool use), the cognitive overhead of manually coordinating prompt interactions becomes unmanageable.
4
The Reproducibility Crisis
Prompt changes are not tracked like code changes. When an output changes, it's hard to know if it's the prompt, the model version, the retrieval index, or sampling parameters. This violates basic ML reproducibility principles.
5
The Governance Gap
Regulated industries (finance, healthcare, legal) need auditable pipelines. A JSON file of prompt strings does not constitute an auditable, versioned artifact. Compliance teams cannot reason about what a prompt-based system will do.

The Academic Foundation

DSPy emerged from research at Stanford NLP Group (Omar Khattab et al., 2023). The key insight was drawing a parallel between:

// The ML Analogy that DSPy is Built On
 Traditional ML                     DSPy LLM Programming
 ─────────────────────────────────────────────────────────────────
 Model weights                  =    LLM weights (frozen or LoRA)
 Learnable parameters           =    Prompt instructions + few-shots
 Loss function                  =    Evaluation metric (exact match, F1, etc.)
 SGD / Adam optimizer           =    DSPy compiler (MIPROv2, BootstrapFewShot)
 Training loop                  =    Optimization loop over dev set
 Model checkpoint               =    Serialized DSPy program (.json)
 Test set evaluation            =    Held-out benchmark evaluation
 Data augmentation              =    Bootstrap: LLM generates synthetic examples
 Hyperparameter search          =    Instruction candidate search
 ─────────────────────────────────────────────────────────────────
 INSIGHT: If we treat prompts as PARAMETERS, we can OPTIMIZE them.

The DSPy Design Principles

📝
Separation of Intent and Implementation
Signatures declare what you want. Modules and optimizers handle how to get it. This mirrors the declarative SQL paradigm — you don't write the query plan, the database optimizer does.
🔬
Empiricism Over Intuition
Every pipeline decision should be backed by measurable improvement on a validation set. DSPy makes empiricism the default workflow, not an afterthought.
🏗️
Composability
Modules compose just like PyTorch layers. Build small, testable components and assemble them into complex pipelines without losing the ability to optimize the whole system end-to-end.
⚠️ When DSPy is NOT the right choice

DSPy has overhead: you need a dev/test dataset, evaluation metrics, and optimization runs (which cost API calls). For simple, one-off prompts or early prototypes, vanilla prompting may be faster. DSPy shines when you need reliability, reproducibility, and systematic improvement in production systems.

Chapter 03 · Core Concepts

DSPy Core Concepts

DSPy has four primary building blocks: Signatures, Modules, Optimizers, and Metrics. Understanding these deeply is essential before writing any pipeline code.

1. Signatures

A Signature is a typed input/output contract for an LLM call. It declares the semantics of what you want without specifying how to achieve it. Think of it as a function prototype for an LLM.

PYTHON · signatures.py
import dspy
from typing import Literal

# ─── Simple QA Signature ───────────────────────────────────────
class BasicQA(dspy.Signature):
    """Answer questions based on provided context. Be factual and concise."""
    context: str = dspy.InputField(desc="Retrieved passages from the knowledge base")
    question: str = dspy.InputField(desc="The user's question")
    answer: str = dspy.OutputField(desc="A concise, factual answer in 1-3 sentences")

# ─── Multi-output Signature ────────────────────────────────────
class AnalyzeDocument(dspy.Signature):
    """Analyze a document and extract structured insights."""
    document: str = dspy.InputField(desc="The document text to analyze")
    summary: str = dspy.OutputField(desc="A 2-sentence summary")
    key_claims: list[str] = dspy.OutputField(desc="Top 3-5 factual claims made")
    sentiment: Literal['positive', 'negative', 'neutral'] = dspy.OutputField()
    confidence: float = dspy.OutputField(desc="Confidence score 0.0-1.0")

# ─── RAG-Specific Signature ────────────────────────────────────
class RAGAnswer(dspy.Signature):
    """Given retrieved context, answer the question faithfully.
    Only use information from the provided context.
    If the answer is not in the context, say 'I cannot find this in the provided documents.'"""
    context: list[str] = dspy.InputField(desc="List of retrieved document chunks")
    question: str = dspy.InputField(desc="The user query")
    answer: str = dspy.OutputField(desc="Faithful answer grounded in context")
    citations: list[str] = dspy.OutputField(desc="Quoted fragments supporting the answer")

# ─── Inline signature shorthand (for simple cases) ─────────────
summarize = dspy.Predict("document: str -> summary: str")
classify  = dspy.Predict("text: str -> label: Literal['spam', 'ham']")
📌 Signature Best Practices

The docstring of a Signature becomes the system instruction. Be precise but not over-constrained. Field desc attributes are used to generate input/output guidance. Type annotations (especially Literal, list, float) enable structured output parsing and validation.

2. Modules

Modules are the execution strategies that wrap signatures. They define how the LLM is invoked — with what prompting strategy. DSPy provides several built-in modules:

ModuleStrategyBest For
dspy.PredictDirect predictionSimple I/O, classification, extraction
dspy.ChainOfThoughtGenerates rationale before answerMulti-step reasoning, math, logic
dspy.ProgramOfThoughtGenerates Python code, executes itQuantitative reasoning, computation
dspy.ReActInterleaved reasoning + tool callingAgent tasks, tool-augmented QA
dspy.MultiChainComparisonGenerates N chains, picks bestHigh-stakes decisions, ensemble
dspy.RetrieveVector search retrievalRAG pipelines
PYTHON · modules.py
import dspy

# ─── Custom Module: composing multiple steps ───────────────────
class HierarchicalRAG(dspy.Module):
    """Multi-hop RAG with query decomposition and answer synthesis."""

    def __init__(self, num_passages: int = 5, num_hops: int = 2):
        super().__init__()
        self.num_hops = num_hops

        # Declare sub-modules (DSPy tracks these for optimization)
        self.decompose = dspy.ChainOfThought("question -> subquestions: list[str]")
        self.retrieve = dspy.Retrieve(k=num_passages)
        self.synthesize = dspy.ChainOfThought(RAGAnswer)

    def forward(self, question: str) -> dspy.Prediction:
        # Step 1: Decompose complex question
        decomposed = self.decompose(question=question)
        subquestions = decomposed.subquestions

        # Step 2: Retrieve passages for each subquestion
        all_passages = []
        for subq in subquestions[:self.num_hops]:
            passages = self.retrieve(query=subq).passages
            all_passages.extend(passages)

        # Remove duplicates while preserving order
        seen = set()
        unique_passages = [
            p for p in all_passages
            if not (p in seen or seen.add(p))
        ]

        # Step 3: Synthesize final answer from all retrieved context
        result = self.synthesize(
            context=unique_passages,
            question=question
        )

        return dspy.Prediction(
            answer=result.answer,
            citations=result.citations,
            context=unique_passages,
            subquestions=subquestions
        )

3. Optimizers (Compilers)

Optimizers are the heart of DSPy's value proposition. They search over the space of possible prompts and few-shot examples to find configurations that maximize your metric.

🔍
BootstrapFewShot
Runs the program on training examples, collects successful traces, and injects them as few-shot examples. Fast, low-cost, great baseline optimizer.
🧬
MIPROv2
Bayesian optimization over instruction candidates + few-shot examples. The most powerful DSPy optimizer. Uses a meta-LLM to propose instruction variants.
⚡
BootstrapFewShotWithRandomSearch
Runs BootstrapFewShot multiple times with different random seeds, picks the best program by dev set score. Good balance of cost and quality.
🏆
COPRO
Coordinate-ascent optimizer that iteratively improves instructions per module using a gradient-free search. Good for instruction-only optimization.

Philosophy Deep Dive

LangChain: The Integration Layer

LangChain's philosophy is connectivity. It provides a massive ecosystem of integrations (100+ LLMs, 50+ vector stores, dozens of document loaders) connected through a composable "chain" abstraction. Its strength is getting started fast. Its weakness is that prompt management remains manual, evaluation requires external tooling (LangSmith), and complex chains become hard to reason about.

LangChain vs DSPy in Practice

LangChain is excellent for prototyping and integration. DSPy is better for production reliability. Many teams use LangChain for data ingestion and retrieval, while using DSPy for the LLM reasoning layer on top.

AutoGen: The Multi-Agent Framework

AutoGen (Microsoft) models LLM applications as conversations between agents. Each agent has a role, a system prompt, and can use tools or call other agents. It excels at complex, dynamic workflows where the path through the pipeline depends on intermediate results. However, it lacks systematic prompt optimization and evaluation tooling.

LangGraph: Stateful Graph Orchestration

LangGraph extends LangChain with a graph-based state machine. Nodes are LLM calls or tool calls; edges are conditional logic. This is powerful for long-horizon agents with complex branching logic (e.g., coding agents, research agents). Like AutoGen, it lacks built-in optimization.

Hybrid Architecture Pattern

In practice, sophisticated teams combine frameworks. Here's a proven enterprise pattern:

PYTHON · hybrid_arch.py
# Layer 1: LangChain for data ingestion + indexing
from langchain.document_loaders import DirectoryLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_community.vectorstores import Chroma

loader = DirectoryLoader("./docs", glob="**/*.pdf")
docs = loader.load()
splitter = RecursiveCharacterTextSplitter(chunk_size=512, chunk_overlap=64)
chunks = splitter.split_documents(docs)
vectorstore = Chroma.from_documents(chunks, embedding_function)

# Layer 2: DSPy for LLM reasoning (wraps LangChain retriever)
import dspy

class ChromaRetriever(dspy.Retrieve):
    def __init__(self, vectorstore, k=5):
        super().__init__(k=k)
        self.store = vectorstore

    def forward(self, query: str):
        results = self.store.similarity_search(query, k=self.k)
        return dspy.Prediction(passages=[r.page_content for r in results])

# Now use DSPy for reasoning with automatic optimization
rag = HierarchicalRAG(retriever=ChromaRetriever(vectorstore))
optimized_rag = MIPROv2().compile(rag, trainset=trainset, metric=metric)

import logging
from pathlib import Path

from dataclasses import dataclass

import chromadb
from chromadb.utils import embedding_functions
from langchain.document_loaders import DirectoryLoader, PyPDFLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter

logger = logging.getLogger("rag.ingestion")

@dataclass
class ChunkConfig:
    chunk_size: int = 512
    chunk_overlap: int = 64
    separators: list = None

    def __post_init__(self):
        if self.separators is None:
            self.separators = ["\n\n", "\n", ". ", " ", ""]

class DocumentIngestionPipeline:
    """
    Production document ingestion pipeline.
    Handles: loading → cleaning → chunking → embedding → indexing.
    """

    def __init__(
        self,
        collection_name: str,
        persist_dir: str = "./chroma_db",
        embedding_model: str = "text-embedding-3-small",
        chunk_config: ChunkConfig = None
    ):
        self.collection_name = collection_name
        self.chunk_config = chunk_config or ChunkConfig()

        # Initialize ChromaDB with persistence
        self.client = chromadb.PersistentClient(path=persist_dir)
        self.ef = embedding_functions.OpenAIEmbeddingFunction(
            model_name=embedding_model
        )
        self.collection = self.client.get_or_create_collection(
            name=collection_name,
            embedding_function=self.ef,
            metadata={"hnsw:space": "cosine"}
        )
        logger.info(f"Initialized collection '{collection_name}'")

    def ingest_directory(self, directory: str, glob: str = "**/*.pdf") -> int:
        """Load all documents from a directory and index them."""
        loader = DirectoryLoader(directory, glob=glob, loader_cls=PyPDFLoader)
        documents = loader.load()
        logger.info(f"Loaded {len(documents)} documents")

        # Chunk documents
        splitter = RecursiveCharacterTextSplitter(
            chunk_size=self.chunk_config.chunk_size,
            chunk_overlap=self.chunk_config.chunk_overlap,
            separators=self.chunk_config.separators
        )
        chunks = splitter.split_documents(documents)
        logger.info(f"Generated {len(chunks)} chunks")

        # Prepare for indexing
        ids, texts, metadatas = [], [], []
        for i, chunk in enumerate(chunks):
            chunk_id = f"chunk_{i:06d}"
            ids.append(chunk_id)
            texts.append(chunk.page_content)
            metadatas.append({
                "source": chunk.metadata.get("source", "unknown"),
                "page": chunk.metadata.get("page", 0),
                "chunk_index": i,
            })

        # Batch upsert to ChromaDB (max 5000 per batch)
        batch_size = 500
        for i in range(0, len(ids), batch_size):
            self.collection.upsert(
                ids=ids[i:i+batch_size],
                documents=texts[i:i+batch_size],
                metadatas=metadatas[i:i+batch_size]
            )
            logger.info(f"Indexed batch {i//batch_size + 1}")

        return len(chunks)

    def query(self, query: str, k: int = 5) -> List[Dict[str, Any]]:
        """Retrieve top-k passages for a query."""
        results = self.collection.query(
            query_texts=[query],
            n_results=k,
            include=["documents", "metadatas", "distances"]
        )
        return [
            {
                "text": doc,
                "metadata": meta,
                "score": 1 - dist  # Convert distance to similarity
            }
            for doc, meta, dist in zip(
                results["documents"][0],
                results["metadatas"][0],
                results["distances"][0]
            )
        ]

from typing import List, Optional

class ChromaDSPyRetriever(dspy.Retrieve):
    """DSPy-compatible retriever backed by ChromaDB."""

    def __init__(self, pipeline: DocumentIngestionPipeline, k: int = 5):
        super().__init__(k=k)
        self.pipeline = pipeline

    def forward(self, query: str, k: Optional[int] = None) -> dspy.Prediction:
        k = k or self.k
        results = self.pipeline.query(query, k=k)
        passages = [r["text"] for r in results]
        metadatas = [r["metadata"] for r in results]
        scores = [r["score"] for r in results]
        return dspy.Prediction(
            passages=passages,
            metadatas=metadatas,
            scores=scores
        )


class HybridRetriever(dspy.Module):
    """Combines dense (vector) + sparse (BM25) retrieval."""

    def __init__(self, dense_retriever, bm25_retriever, k: int = 5, alpha: float = 0.7):
        super().__init__()
        self.dense = dense_retriever
        self.bm25 = bm25_retriever
        self.k = k
        self.alpha = alpha  # Weight: 0=pure BM25, 1=pure dense

    def forward(self, query: str) -> dspy.Prediction:
        dense_results = self.dense(query=query, k=self.k*2)
        bm25_results = self.bm25(query=query, k=self.k*2)

        # Reciprocal Rank Fusion (RRF)
        passage_scores = {}
        for rank, passage in enumerate(dense_results.passages):
            passage_scores[passage] = passage_scores.get(passage, 0) + \
                self.alpha * 1.0 / (rank + 60)
        for rank, passage in enumerate(bm25_results.passages):
            passage_scores[passage] = passage_scores.get(passage, 0) + \
                (1 - self.alpha) * 1.0 / (rank + 60)

        top_passages = sorted(passage_scores, key=passage_scores.get, reverse=True)[:self.k]
        return dspy.Prediction(passages=top_passages)

import dspy
from typing import List

# ─── Signatures ───────────────────────────────────────────────────

class QueryRewrite(dspy.Signature):
    """Rewrite a user query into multiple search-optimized sub-queries.
    Generate queries that together cover all aspects of the original question."""
    original_query: str = dspy.InputField()
    rewritten_queries: List[str] = dspy.OutputField(
        desc="2-4 distinct search queries covering different aspects"
    )

class PassageRelevance(dspy.Signature):
    """Assess whether a passage is relevant to a question. Be strict."""
    passage: str = dspy.InputField()
    question: str = dspy.InputField()
    is_relevant: bool = dspy.OutputField()
    relevance_reason: str = dspy.OutputField(desc="One sentence explanation")

class FinalAnswer(dspy.Signature):
    """Synthesize a comprehensive, faithful answer from retrieved context.

    Rules:
    - Only use information from the provided context
    - If context doesn't contain the answer, say so explicitly
    - Always cite specific passages that support your answer
    - Be precise and avoid speculation"""
    context: List[str] = dspy.InputField(desc="Top retrieved passages")
    question: str = dspy.InputField()
    answer: str = dspy.OutputField(desc="Comprehensive, faithful answer")
    citations: List[str] = dspy.OutputField(desc="Exact quotes from context")
    confidence: float = dspy.OutputField(desc="Answer confidence 0.0-1.0")

# ─── Main RAG Pipeline Module ─────────────────────────────────────

class EvaluationDrivenRAG(dspy.Module):
    """
    Multi-hop RAG pipeline with query rewriting, reranking, and
    faithful answer synthesis. Designed for DSPy optimization.
    """

    def __init__(self, retriever, num_passages: int = 5, max_hops: int = 2):
        super().__init__()
        self.retriever = retriever
        self.num_passages = num_passages
        self.max_hops = max_hops

        # Sub-modules (all optimizable by DSPy)
        self.query_rewriter = dspy.ChainOfThought(QueryRewrite)
        self.relevance_filter = dspy.Predict(PassageRelevance)
        self.synthesizer = dspy.ChainOfThought(FinalAnswer)

    def forward(self, question: str) -> dspy.Prediction:
        # Step 1: Query rewriting for better recall
        rewrite_result = self.query_rewriter(original_query=question)
        queries = rewrite_result.rewritten_queries
        if question not in queries:  # Always include original
            queries = [question] + queries[:self.max_hops - 1]

        # Step 2: Retrieve + deduplicate across all queries
        all_passages = []
        seen = set()
        for query in queries[:self.max_hops]:
            result = self.retriever(query=query)
            for p in result.passages:
                if p not in seen:
                    all_passages.append(p)
                    seen.add(p)

        # Step 3: Relevance filtering (keeps top N relevant passages)
        relevant_passages = []
        for passage in all_passages:
            relevance = self.relevance_filter(
                passage=passage,
                question=question
            )
            if relevance.is_relevant:
                relevant_passages.append(passage)
            if len(relevant_passages) >= self.num_passages:
                break

        # Fallback: use all retrieved if none pass filter
        context = relevant_passages or all_passages[:self.num_passages]

        # Step 4: Faithful synthesis
        synthesis = self.synthesizer(context=context, question=question)

        return dspy.Prediction(
            answer=synthesis.answer,
            citations=synthesis.citations,
            confidence=synthesis.confidence,
            context=context,
            queries_used=queries
        )

class JSONFormatter(logging.Formatter):
    """Outputs log records as JSON lines for log aggregation systems."""

    def format(self, record: logging.LogRecord) -> str:
        log_obj = {
            "timestamp": self.formatTime(record),
            "level": record.levelname,
            "logger": record.name,
            "message": record.getMessage(),
            "module": record.module,
        }
        if hasattr(record, "extra"):
            log_obj.update(record.extra)
        if record.exc_info:
            log_obj["exception"] = self.formatException(record.exc_info)
        return json.dumps(log_obj)

def setup_logging(level: str = "INFO", json_output: bool = False):
    handlers = [logging.StreamHandler()]
    if json_output:
        handlers[0].setFormatter(JSONFormatter())
    logging.basicConfig(
        level=getattr(logging, level.upper()),
        handlers=handlers
    )

def trace_dspy_call(func: Callable) -> Callable:
    """Decorator to log DSPy module calls with latency and token usage."""
    @wraps(func)
    def wrapper(*args, **kwargs):
        logger = logging.getLogger(func.__module__)
        start = time.perf_counter()
        try:
            result = func(*args, **kwargs)
            elapsed = (time.perf_counter() - start) * 1000
            logger.info(f"[TRACE] {func.__name__} completed in {elapsed:.1f}ms")
            return result
        except Exception as e:
            logger.error(f"[TRACE] {func.__name__} failed: {e}", exc_info=True)
            raise
    return wrapper

from dspy.teleprompt import BootstrapFewShot, BootstrapFewShotWithRandomSearch

def run_bootstrap_optimization(
    pipeline,
    trainset: list,
    metric,
    max_bootstrapped_demos: int = 4,
    max_labeled_demos: int = 2,
    num_candidate_programs: int = 8,
    save_path: str = "./models/bootstrapped_rag.json"
):
    """
    Run BootstrapFewShotWithRandomSearch optimization.

    This optimizer:
    1. Runs program on trainset with teacher LM
    2. Collects (input → output) traces where metric is satisfied
    3. Samples different combinations of these traces as few-shots
    4. Picks the combination with best devset score
    """
    optimizer = BootstrapFewShotWithRandomSearch(
        metric=metric,
        max_bootstrapped_demos=max_bootstrapped_demos,
        max_labeled_demos=max_labeled_demos,
        num_candidate_programs=num_candidate_programs,
        num_threads=8  # Parallel evaluation
    )

    # Split trainset into train / dev
    split = int(len(trainset) * 0.8)
    train, dev = trainset[:split], trainset[split:]

    optimized = optimizer.compile(
        pipeline,
        trainset=train,
        valset=dev
    )

    optimized.save(save_path)
    return optimized

import dspy
from dspy.teleprompt import MIPROv2
import logging

logger = logging.getLogger("optimization.mipro")

def run_mipro_optimization(
    pipeline,
    trainset: list,
    devset: list,
    metric,
    teacher_model: str = "openai/gpt-4o",
    student_model: str = "openai/gpt-4o-mini",
    num_candidates: int = 10,
    num_trials: int = 20,
    max_bootstrapped_demos: int = 4,
    save_path: str = "./models/mipro_rag.json"
):
    """
    Run MIPROv2 (MIPRO version 2) optimization.

    MIPROv2 uses Bayesian Optimization (TPE) to search over:
    - N instruction candidates per module (generated by teacher LM)
    - M few-shot example sets bootstrapped from training data

    The optimization objective: maximize metric(program(example), example)
    averaged over the devset.
    """
    logger.info(f"Starting MIPROv2 optimization")
    logger.info(f"Trainset: {len(trainset)}, Devset: {len(devset)}")
    logger.info(f"Candidates: {num_candidates}, Trials: {num_trials}")

    # Configure teacher LM for bootstrap + instruction generation
    teacher_lm = dspy.LM(teacher_model, max_tokens=4096)
    student_lm = dspy.LM(student_model, max_tokens=2048)

    optimizer = MIPROv2(
        metric=metric,
        auto="medium",  # 'light', 'medium', 'heavy'
        num_candidates=num_candidates,
        num_threads=8,
        prompt_model=teacher_lm,  # Used to generate instruction candidates
        task_model=student_lm,    # Used to evaluate candidates
        verbose=True
    )

    with dspy.context(lm=student_lm):
        optimized = optimizer.compile(
            pipeline,
            trainset=trainset,
            num_trials=num_trials,
            max_bootstrapped_demos=max_bootstrapped_demos,
            max_labeled_demos=2,
            eval_kwargs={"num_threads": 8, "display_progress": True}
        )

    optimized.save(save_path)
    logger.info(f"Optimized program saved to {save_path}")
    return optimized

    data_path: str = "./data/qa_dataset.jsonl",
    max_examples: int = None
) -> List[dspy.Example]:
    """Load QA dataset as DSPy Examples."""
    examples = []
    with open(data_path) as f:
        for line in f:
            obj = json.loads(line.strip())
            if obj.get("split") != split:
                continue
            example = dspy.Example(
                question=obj["question"],
                answer=obj["answer"],  # Ground truth
                context=obj.get("context", []),
            ).with_inputs("question")  # Tell DSPy 'question' is the only input field
            examples.append(example)

    if max_examples:
        examples = examples[:max_examples]
    return examples


class SyntheticDataGenerator:
    """
    Generates synthetic QA pairs from documents using a strong LLM.
    Useful for cold-start when no human annotations exist.
    """

    def __init__(self, generator_lm: str = "openai/gpt-4o"):
        self.generate = dspy.Predict(
            "passage: str -> question: str, answer: str, difficulty: str"
        )
        self.lm = dspy.LM(generator_lm)

    def generate_from_passages(self, passages: List[str], n_per_passage: int = 3):
        """Generate n QA pairs per passage."""
        examples = []
        with dspy.context(lm=self.lm):
            for passage in passages:
                for _ in range(n_per_passage):
                    result = self.generate(passage=passage)
                    examples.append(dspy.Example(
                        question=result.question,
                        answer=result.answer,
                        difficulty=result.difficulty,
                        source_passage=passage
                    ).with_inputs("question"))
        return examples

"""
DSPy metric functions. These are the optimization objectives.
A metric takes (example, prediction, trace=None) and returns a float or bool.
"""
import re
from typing import Optional
import dspy

# ─── 1. Exact Match (baseline) ─────────────────────────────────
def exact_match(example, pred, trace=None) -> bool:
    gold = example.answer.strip().lower()
    predicted = pred.answer.strip().lower()
    return gold == predicted

# ─── 2. Token F1 (partial credit) ──────────────────────────────
def token_f1(example, pred, trace=None) -> float:
    def tokens(text): return set(re.findall(r'\w+', text.lower()))
    gold = tokens(example.answer)
    predicted = tokens(pred.answer)
    if not gold or not predicted: return 0.0
    precision = len(gold & predicted) / len(predicted)
    recall = len(gold & predicted) / len(gold)
    if precision + recall == 0: return 0.0
    return 2 * precision * recall / (precision + recall)

# ─── 3. LLM-as-Judge (semantic faithfulness) ──────────────────
class FaithfulnessJudge(dspy.Signature):
    """Evaluate if the predicted answer is faithful to the context and correct.
    Be strict: partial answers should score below 0.7."""
    context: list = dspy.InputField()
    question: str = dspy.InputField()
    gold_answer: str = dspy.InputField()
    predicted_answer: str = dspy.InputField()
    faithfulness_score: float = dspy.OutputField(desc="0.0=hallucinated, 1.0=perfectly faithful")
    correctness_score: float = dspy.OutputField(desc="0.0=wrong, 1.0=exactly right")

judge_module = dspy.Predict(FaithfulnessJudge)

def llm_judge_metric(example, pred, trace=None) -> float:
    """LLM-as-Judge metric: weighted combination of faithfulness + correctness."""
    try:
        result = judge_module(
            context=getattr(pred, "context", []),
            question=example.question,
            gold_answer=example.answer,
            predicted_answer=pred.answer
        )
        faithfulness = float(result.faithfulness_score)
        correctness = float(result.correctness_score)
        return 0.4 * faithfulness + 0.6 * correctness
    except:
        return 0.0

# ─── 4. Composite RAG Metric (recommended for production) ──────
def rag_metric(example, pred, trace=None) -> float:
    """
    Composite metric for RAG optimization.
    Combines token F1, faithfulness (LLM judge), and citation quality.
    """
    f1 = token_f1(example, pred)
    faithfulness = llm_judge_metric(example, pred)

    # Citation quality bonus
    citations = getattr(pred, "citations", [])
    citation_score = min(len(citations) / 2, 1.0) if citations else 0.0

    # During optimization (trace is not None), use faster metric
    if trace is not None:
        return f1 >= 0.5  # Binary for bootstrap

    return 0.5 * f1 + 0.35 * faithfulness + 0.15 * citation_score

Chapter 11 · Enterprise

Best Practices for Enterprise LLM Engineering

Enterprise LLM systems require more than good prompts. They need reproducibility, governance, observability, and systematic improvement loops. This chapter covers battle-tested patterns.

1. Evaluation-Driven Development (EDD)

Never modify a production LLM pipeline without first measuring its current performance. The EDD workflow mirrors Test-Driven Development:

1
Write the metric first
Before writing any code, define what "good" looks like quantitatively. What F1 score is acceptable? What hallucination rate is tolerable? These become your metric functions.
2
Build a representative evaluation set
Minimum 100 human-annotated examples covering edge cases, difficult questions, and "cannot answer" scenarios. Never optimize and evaluate on the same set.
3
Establish a baseline score
Run your unoptimized pipeline and record scores. This is your benchmark. Every change must beat the baseline to ship.
4
Optimize, evaluate, iterate
Run optimization. Score on devset. Only promote to production if test set scores improve. Track all scores in your experiment registry.

import hashlib, json, datetime

def version_program(program, metadata: dict) -> str:
    """Generate a content-addressable version ID for a DSPy program."""
    # Save program state
    state = program.dump_state()
    content = json.dumps(state, sort_keys=True)
    version_hash = hashlib.sha256(content.encode()).hexdigest()[:12]
    timestamp = datetime.datetime.utcnow().isoformat()

    versioned = {
        "version": version_hash,
        "created_at": timestamp,
        "metadata": metadata,  # model, optimizer, metric scores
        "program_state": state
    }
    path = f"./models/program_{version_hash}.json"
    with open(path, "w") as f:
        json.dump(versioned, f, indent=2)
    return version_hash
